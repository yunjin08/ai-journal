# AI Journal

A personal journal for tracking problems I solve as a software engineer — with one rule: I write my own thinking first, before touching any AI tool.

Every entry has two stages. Stage 1 is my approach: the problem, how I thought about it, what I would do. Once I commit that, it's locked forever. Stage 2 is what happened after: what the AI said, how it differed, and what I actually learned.

The lock is the point. It exists to stop the habit of reaching for AI the moment a problem gets hard. The discomfort of having to write something before you're allowed to look anything up is where the learning happens.

This is not an anti-AI tool. It's an anti-shortcut tool.
